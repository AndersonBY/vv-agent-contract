# Typed Event Stream

The framework exposes one event surface: the typed `RunEvent` journal. Runtime
lifecycle producers, tool execution, memory, checkpoints, configured children,
and model streaming all enter the same journal before an optional host observer
is notified. Event observation never classifies the task or changes model,
tool, completion, budget, cancellation, approval, or after-cycle decisions.

## One Public Surface

`RunConfig.stream` observes `RunEvent` values. `RunHandle` exposes the same
typed sequence, and `RunEventStore` provides replay under its declared failure
policy. There is no public runtime-log callback and no public provider-payload
callback.

Observer failure is isolated from execution. A configured event store remains
separate: fail-open stores report and continue, while an explicitly selected
fail-closed store may stop the run according to the storage contract.

## Private LLM Adapter Boundary

Provider SDK payloads remain private to an LLM adapter. The adapter validates
and projects only these source discriminators into framework-owned events:

| Adapter source | Typed RunEvent type | Meaning |
| --- | --- | --- |
| `assistant_delta` | `assistant_delta` | Visible assistant text generated by the model |
| `reasoning_delta` | `reasoning_delta` | Private reasoning text generated by the model |
| `tool_call_started` | `model_tool_call_started` | The model started generating one tool call |
| `tool_call_progress` | `model_tool_call_progress` | The model advanced generation of one tool call |

`model_tool_call_started` is distinct from `tool_call_started`: the former is
model-generation telemetry, while the latter means the runtime executor has a
complete call and is entering execution.

Unknown, invalid, or lifecycle-shaped provider payloads are dropped at the
adapter boundary. They cannot fabricate run, cycle, tool-execution, approval,
handoff, or terminal events, and no public observer receives the raw payload.
Framework `run_id`, `trace_id`, `session_id`, `agent_name`, parent identity, and
positive `cycle_index` always override provider values.

## Typed Stream Fields

`assistant_delta` contains `delta` plus optional cumulative `content_chars` and
`estimated_tokens`. `reasoning_delta` contains `delta` plus optional cumulative
`reasoning_chars` and `estimated_tokens`.

Both model tool-call events contain non-empty `tool_call_id` and `tool_name`,
plus optional `tool_call_index`, `arguments_chars`, and `estimated_tokens`.
Numeric fields are non-negative JSON-safe integers no greater than
`9007199254740991`. Unavailable metrics are omitted and never reconstructed as
zero.

## Diagnostics

Runtime diagnostics use the typed `diagnostic` event. It requires:

- `level`: `debug`, `info`, `warning`, or `error`;
- `code`: a stable non-empty machine-readable code;
- `details`: a JSON object containing task-neutral diagnostic values.

The `details` object is the explicitly open extension map for this event. It
must contain JSON values and cannot override event identity or canonical
fields. Diagnostic events are observational: hosts must not infer run state,
tool execution, approval, or completion from them. Semantic state always comes
from the corresponding lifecycle event.

## Top-Level And Child Runs

Top-level and configured child runs use the same typed wire. Child producers
create framework-owned child run/session/agent identity and include
`parent_run_id`. Private adapter receipts and provider fields never enter the
public event. Child event forwarding preserves the original event id and
timestamp instead of serializing through a raw callback.

## Ordering And Delivery

For one production step, the runtime records the typed event before calling the
optional observer. Local order is preserved, but only event ids provide
identity. Durable replay and deduplication require `RunEventStore` and, where
configured, the checkpoint event outbox.

Disabling an observer does not disable the internal typed journal and does not
add prompts, messages, tools, denials, retries, or completion rules.

## App Server Projection

Reasoning deltas and diagnostics are private telemetry by default and are not
projected as visible assistant messages. Model tool-call progress may be
projected as a tool-call delta. Actual execution continues to use
`tool_call_started` and `tool_call_completed`. Hosts may suppress telemetry
notifications without changing the underlying run.

## Executable Evidence

`llm_stream_projection.json` locks the private adapter mapping, identity,
drop policy, and single typed observer surface. `run_events.jsonl` locks all
current wire variants, including diagnostics. `configured_sub_agent.json`
locks typed child forwarding. Both implementations must test real Runner
producers, not only a private mapping helper.
