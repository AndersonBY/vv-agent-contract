# Stream Event Projection

Contract `0.7.x` defines one typed projection for model stream callbacks at the
top-level Runner and configured child-agent boundaries. The projection is
runtime observability. It does not classify a task, inspect answer quality, or
change model, tool, completion, budget, cancellation, or approval decisions.

## Two Surfaces

The SDKs expose two deliberately different surfaces:

1. The raw model callback carries provider-adapter payloads. It is synchronous,
   best-effort, and at-least-once. It has no durable cursor or delivery claim.
2. `RunEvent` is the typed runtime journal. It owns framework identity, event
   ids, cycle identity, serialization, and optional event-store persistence.

A raw callback is not an event store. Applications that require replay use a
`RunEventStore` and its declared failure policy.

## Source To Wire Mapping

Only these raw source discriminators enter the typed event journal:

| Raw source event | Typed RunEvent type | Meaning |
| --- | --- | --- |
| `assistant_delta` | `assistant_delta` | Visible assistant text generated by the model |
| `reasoning_delta` | `reasoning_delta` | Private reasoning text generated by the model |
| `tool_call_started` | `model_tool_call_started` | The model started generating one tool call |
| `tool_call_progress` | `model_tool_call_progress` | The model advanced generation of one tool call |

`model_tool_call_started` is intentionally distinct from the existing
`tool_call_started`. The former is model-generation telemetry and may contain
only cumulative argument length. The latter means the runtime executor has a
complete call and is about to execute it.

Unknown source events, lifecycle-shaped source events, and invalid known
payloads are not converted into `RunEvent`. An explicitly configured raw
observer still receives the original source payload. This keeps adapters
extensible without allowing an untrusted model callback to fabricate a run,
cycle, tool-execution, approval, handoff, or terminal event.

## Typed Fields

All four typed stream events use the framework-owned `run_id`, `trace_id`,
`session_id`, `agent_name`, and positive `cycle_index`. Source values cannot
override them.

`assistant_delta` contains:

- `delta`: the visible text fragment.
- optional `content_chars`: cumulative visible character count.
- optional `estimated_tokens`: cumulative adapter estimate.

`reasoning_delta` contains:

- `delta`: the private reasoning fragment.
- optional `reasoning_chars`: cumulative reasoning character count.
- optional `estimated_tokens`: cumulative adapter estimate.

Both model tool-call events contain:

- `tool_call_id`: non-empty model tool-call identity.
- optional `tool_call_index`: non-negative position in the model response.
- `tool_name`: the adapter's current non-empty function name.
- optional `arguments_chars`: cumulative generated argument character count.
- optional `estimated_tokens`: cumulative adapter estimate.

Numeric fields are non-negative JSON-safe integers no greater than
`9007199254740991`. Optional metrics are omitted when unavailable and are never
reconstructed from zero or another field.

## Top-Level And Child Runs

Top-level and configured child runs use the same source-to-payload mapping and
validation. The scope changes identity only:

- A top-level event uses the current Runner identity.
- A child event uses the framework-created child run/session/agent identity,
  includes `parent_run_id`, and retains the canonical child source payload in
  metadata for compatibility.

The runtime inserts the current cycle before projection. Model-supplied cycle,
run, trace, session, agent, parent, event, status, and terminal fields are not
trusted. Canonical child receipt and sequence markers are internal validation
data and never appear in the public event.

## Ordering And Failure Isolation

For one callback invocation, the runtime records the typed event before calling
the optional raw observer. Callback order is preserved in the local event
journal. Event ids provide identity; callback order alone does not provide an
exactly-once or durable-delivery guarantee.

An absent, dropped, closed, raising, or panicking observer cannot suppress a
typed event or alter runtime decisions and terminal projection. Observer
failure is isolated. A configured event store remains separate: fail-open
stores report and continue, while an explicitly selected fail-closed store may
stop the run according to the existing storage contract.

Disabling an observer does not disable the internal typed journal. It also
does not add prompts, messages, tools, denials, retries, or completion rules.

## App Server Projection

Reasoning deltas are private telemetry and are not projected as visible agent
messages. Model tool-call progress may be projected as a tool-call delta.
Actual tool execution continues to use `tool_call_started` and
`tool_call_completed`. Hosts may choose not to publish any stream telemetry to
clients without changing the underlying run.

## Executable Evidence

`stream_projection_v1.json` locks raw-to-typed names, fields, identity, order,
unknown-event rejection, observer isolation, and the distinction between model
generation and tool execution. `run_events_v1.jsonl` locks the new wire
variants. `configured_sub_agent_v1.json` locks the matching child projection.

Both implementations must test the public Runner against the synthetic source
sequence, not only a private mapping helper. The full repository gates and the
central cross-repository workflow remain required before support is marked
`verified`.
